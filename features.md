# 功能模組需求

## 跨功能共用規格

### 員工選擇與離職處理

凡是需要選擇員工的欄位（Owner、Shift、操作者等），統一使用**下拉選單從員工表選擇**，不允許自由輸入。

- 員工表使用 `is_active` 欄位區分在職/停用
- **下拉選單**：僅顯示 `is_active = true` 的員工
- **歷史紀錄**：已停用員工的姓名仍正常顯示，不受影響
- 適用功能：Feature 1、3、4、10、11 等所有涉及人員選擇的模組

### 公告展示型功能共用規格

Feature 7、8、9、12、13 屬公告展示型，共用以下規格：

- 管理者可使用**區塊式版面編輯器**自訂版面（文字、圖片、影片、連結、行事曆區塊）
- **圖片**：上傳後顯示，儲存於 VM 本地磁碟
- **影片**：上傳後須可直接在頁面內播放

### 設備主檔共用（Feature 1 ↔ Feature 4）

Feature 4（運轉交接）的 Equipment 欄位（R-03 待確認）可能需關聯 Feature 1（Equipment Tracking）的設備主檔。

- 若採**下拉選單**關聯 Feature 1 設備表，需注意權限差異：Feature 1 新增設備限「管理者」，但 Feature 4 所有使用者可新增交接紀錄 → 若遇到設備表中不存在的設備，使用者無法自行新增
- 若採**自由輸入**，則兩功能的設備資料無法互通，統計與查詢困難
- **待確認**：X-01（見 [pending_confirmations.md](pending_confirmations.md)）

### Alarm 與 Abnormal Review 的關聯（Feature 2 ↔ Feature 10）

Feature 2（Alarm）與 Feature 10（Abnormal Review）的系統分類皆為 WTS / EI / GC / HVAC，兩者可能需要數據串接。

- AB-03 待確認：Alarm 是否可直接轉為 Abnormal 登記
- 若可轉移，需定義：**欄位映射規則**（Alarm 的哪些欄位對應 Abnormal 的哪些欄位）、**operation_logs 記錄方式**（跨模組操作如何記錄）
- **待確認**：X-02（見 [pending_confirmations.md](pending_confirmations.md)）

### 運轉交接與異常事件的關聯（Feature 4 ↔ Feature 10）

Feature 4（運轉交接）與 Feature 10（Abnormal Review）在業務流程上可能相關：異常事件可能需要出現在交接紀錄中，或反之。

- **待確認**：AB-06、X-03（見 [pending_confirmations.md](pending_confirmations.md)）

### 出勤與每日工項的關聯（Feature 3 ↔ Feature 11）

Feature 3（出勤紀錄）與 Feature 11（每日工項）皆涉及每日人員活動紀錄，主管視角可能需要同時查看。

- 兩功能的「人員」資料是否同源？出工狀態與出勤狀態是否需一致性檢查？
- **待確認**：X-04（見 [pending_confirmations.md](pending_confirmations.md)）

### 跨廠區整合模式（Feature 2、4、11）

Feature 2（Alarm）、Feature 4（運轉交接）、Feature 11（每日工項）皆為 A 廠有既有 Web/DB、B 廠用 Excel 的整合模式。

- **Feature 2** 已識別 B 廠網路連通性問題（A-08），並提出轉接伺服器方案
- **Feature 4 和 Feature 11** 採相同跨廠整合模式，但尚未確認是否有相同的 B 廠網路限制
- **待確認**：X-05（見 [pending_confirmations.md](pending_confirmations.md)）

### Feature 3 — 出勤紀錄（排班）

- **性質**：每日出勤登記（非打卡考勤），全體 Facilities 人員自行記錄
- **與 HR 系統無關**，純部門內部管理用途
- **資料來源**：Excel → 新建 MSSql DB

#### 現有 Excel 格式

- 橫軸 = 日期，縱軸 = 人員姓名
- 格子內容 = **工時數字** 或 **Training** 等文字標記

#### 功能

| 功能           | 說明                                                    |
| -------------- | ------------------------------------------------------- |
| 每日出勤登記   | 個人自行登入後記錄當日出勤（時數 / Training / **請假** / 其他） |
| 出勤總覽       | 主管可一目了然查看全員某段期間的出勤狀態（類似 Excel 表格視圖） |
| 個人紀錄查詢   | 查看自己的歷史出勤紀錄                                  |

#### 權限

| 角色         | 權限                                |
| ------------ | ----------------------------------- |
| 一般使用者   | 登記自己的出勤 + 查看自己的紀錄     |
| 管理者       | 查看全員出勤總覽                    |

#### UI 方向

以**萬年曆（Calendar）**形式呈現，使用 Ant Design Calendar 元件：

- **個人視角**：日曆上每格顯示自己當日的出勤狀態（時數 / Training / 空白）
- **主管視角**：點選某一天可展開查看全員當日出勤狀態，或切換為月曆總覽

#### 請假功能延伸

業主需求包含**請假登記**，功能性質接近輕量化人資出勤系統，需釐清範圍邊界：

- 請假類型：特休、事假、病假、公假等（具體類型待確認 → O-05）
- 請假流程：**無需審核**，純記錄用途（O-06 已確認）
- 與 HR 系統關係：假別額度是否需同步 HR 系統，或純記錄用途（待確認 → O-07）

- **待確認**：O-01 格子內容有哪些類型（時數/Training/請假/還有其他？）、O-02 是否需要事先排班功能還是純登記、O-03 是否需要統計報表（月工時加總等）、O-04 是否可補登/修改過去日期的紀錄、O-05 請假類型有哪些（特休/事假/病假/公假等）、O-07 假別額度是否需同步 HR 系統或純記錄
