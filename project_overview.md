# 專案概述

為某半導體製程商 Facilities 部門建立以 **SharePoint 為入口**、**Custom Web App 為主體**的管理平台，整合廠務日常管理的 13 個功能模組。另有 9 項功能以 URL 跳轉連結至既有系統。

## 時程

預期 **2 個月**完成（MVP 範圍待確認）

---

## 組織與使用者

| 項目         | 內容                        |
| ------------ | --------------------------- |
| 部門         | Facilities（廠務）          |
| 使用者規模   | 百人級別                    |
| 使用者角色   | 一般使用者、管理者（Admin） |
| IT 支援      | 無專業 IT 支援，自行開發    |

## 使用者與權限管理

獨立的**系統管理頁面**，包含三個模組：

| 模組         | 說明                                                  |
| ------------ | ----------------------------------------------------- |
| 員工表管理   | 新增、編輯、停用員工                                  |
| 角色指派     | 設定一般使用者 / 管理者                               |
| 功能權限矩陣 | 列 = 員工，欄 = 各功能模組，格子 = 開/關（預設全開） |

| 角色         | 說明                                       |
| ------------ | ------------------------------------------ |
| 一般使用者   | 依功能權限矩陣存取被授權的功能             |
| 管理者       | 所有功能 + 系統管理頁（員工/權限設定）     |

### 權限分層說明

系統權限分為三層：

| 層級             | 控制方式                 | 說明                                                                 |
| ---------------- | ------------------------ | -------------------------------------------------------------------- |
| 功能存取權限     | 功能權限矩陣（開/關）   | 控制使用者可否進入某功能模組，預設全開                               |
| 功能內操作權限   | 角色（一般/管理者）      | 控制功能內特定操作（如 Feature 1 新增設備限管理者、Feature 6 建立稽核排程限管理者） |
| 數據級權限       | 後端邏輯                 | 部分功能限制使用者只能操作自己的資料（如 Feature 3 一般使用者只能查看自己的出勤） |

> **注意**：「預設全開」指功能權限矩陣的存取開關預設為開啟，不代表所有角色擁有功能內的全部操作權限。功能內操作權限仍依角色區分。

### 刪除策略

- **員工表**：採軟刪除（`is_active` 欄位），停用後歷史紀錄保留顯示
- **operation_logs**：只寫不刪
- **其他業務資料**：統一刪除策略待確認（⏳ SEC-06），建議採軟刪除（`is_deleted` + `deleted_at`）以保留審計軌跡

---

## 操作 Log（跨功能）

所有寫入操作統一記錄至 `operation_logs` 資料表，由後端 middleware 處理，只寫不刪。管理者可依功能、使用者、時間篩選查詢。

## 風險紀錄

| 風險             | 說明                                                       |
| ---------------- | ---------------------------------------------------------- |
| 時程過緊         | 2個月 × 13個功能 × 無IT支援，需明確定義 MVP 範圍           |
| 資料來源分散     | Excel / 既有DB / 新建DB 並存，整合複雜度高                 |
| 跨廠區整合       | Alarm、每日工項需整合A/B兩廠，DB結構一致性是關鍵風險       |
| 功能待確認比例高 | Feature 5、7、8、12 整體未定，影響開發排程估算             |
